GitHub Traceability Action
Smart GitHub action tool for analyzing code dependency and file traceability.

Overview: A GitHub Action tool designed to perform traceability analysis on code repositories. The action is powered by the COMET library, a machine learning model capable of identifying links between source code files, documentation, and requirements.

Problem Addressed: Traditional traceability approaches often fall short by missing dynamic program data and nuanced relationships between artifacts, making it difficult for developers to maintain and debug projects efficiently.

Solution Details: COMET, a Hierarchical Bayesian Network, intelligently combines static analysis, version history, and developer-provided hints to generate highly accurate traceability links. Integrating this into GitHub via our tool "T-Miner" means teams can run traceability evaluations automatically after each repository push.

Impact: Developers gain immediate insights into the most affected areas of their project with each code update, helping prioritize testing efforts and speeding up the debugging process dramatically. This reduces manual investigation time by over 95%, improving both code quality and developer velocity.

Tools Used: Python, JUnit, Machine Learning Libraries

Campus Network Protocol and Course Registration Tool
Custom wireless protocol and registration accelerator that helps students secure course registrations.

Overview: Built a custom wireless network protocol and course registration tool that allowed users to send more requests quickly through the campus wireless network during peak periods.

Problem Addressed: Traditional wireless protocols focused heavily on data loss protection, which slowed down request rates. During class registration windows, server overloads would lock out new users entirely, preventing fair access to registration resources.

Solution Details: Designed a stripped-back wireless reliability mechanism relying on close proximity, allowing devices to prioritize speed over data loss protection. This enabled faster request bursts at critical moments, helping users contact the server before it became overwhelmed and locked out new requests. Once connected, students could remain first in line to register for their courses.

Evolution: This tool evolved from a simple school project in wireless networking, which I adapted after realizing its practical benefit for campus registration scenarios.

Impact: Provided students a tactical advantage in high-demand registration periods, improving access to needed courses by connecting faster than peers using standard wireless setups.

Tools Used: Java, Python, HTML, SQL
